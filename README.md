# STM32F4_Event_LED_Blink
Event-driven LED blink example for STM32F4 using EXTI and WFE, demonstrating low-power event handling.

## What is an Event?

An event is a signal generated by hardware or software that indicates something has occurred, such as a button press or a timer reaching a certain value. In STM32 microcontrollers, events can be monitored using the EXTI (External Interrupt/Event Controller) system.

Purpose of Using Events

The main purpose of using events is to allow the CPU to wait efficiently for certain conditions without continuously polling. This helps reduce power consumption and frees the CPU to perform other tasks or enter a low-power sleep mode (e.g., using WFE – Wait For Event instruction).

## Difference Between Events and Interrupts

While events and interrupts are related, they are not the same:

Interrupts immediately pause the CPU and execute an Interrupt Service Routine (ISR) when a specific condition occurs. They are suitable for tasks that require immediate response.

Events do not automatically trigger code execution. Instead, they can wake the CPU from sleep or set flags that the program can check. Events are ideal for low-power monitoring where immediate execution is not strictly required.
